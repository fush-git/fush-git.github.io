再定义几个基础函数。

旋转

rotate 操作的本质是把某个给定节点上移一个位置，并保证二叉搜索树的性质不改变。
在 Splay 中，旋转操作分为左旋(Zag) 和右旋(Zig)（图上节点是编号）。

我们来模拟一下右旋的操作（红色是要删除的，蓝色是更改后的）。

这样就完成了一次旋转。
而在实现中，我们会把 zag 和 zig 写在一起。
这里的 rotate(x) 表示将 xxx 旋转到其父节点的位置。

splay

splay(x) 表示把 xxx 节点旋转到根节点。
我们可以简单分成 3 种：

zag/zig

很简单，对 xxx 做一次 rotate 即可（图就不放了）。

zag-zig/zig-zag

即 get(fa(x)) != get(x)。
对 xxx 做两次旋转即可。

zag-zag/zig-zig

即 get(fa(x)) == get(x)，这个时候只旋转 xxx 是不对的。
我们要先旋转 fa(x) 再旋转 xxx（双旋）。

为什么我们要弄出双旋呢？

因为双旋会减少一层，明显会更优。

证明

这段内容来自博客mr_spade。

为了方便描述，定义如下内容：
\qquad我们用 TTT 表示一棵完整的 Splay，并（不严谨地）用 ∣T∣\left|T\right|∣T∣ 表示 TTT 的节点数目。
\qquad如无特殊说明，小写英文字母（如 xxx，yyy，yyy）表示 TTT 的一个节点，x∈Tx \in Tx∈T 表示节点 xxx 在 TTT 中。
\qquad并（不严谨地）用 ∣x∣\left|x\right|∣x∣ 表示以节点为根的子树的大小。
\qquad我们默认 x′x'x′ 代表节点 xxx 在经过了上下文中描述的操作以后的状态，因此对应的 xxx 代表之前的状态。
\qquad我们用 Φ(T)\Phi(T)Φ(T) 表示整棵 TTT 的势能函数，ϕ(x)\phi(x)ϕ(x) 则表示节点对 TTT 贡献的势能。

先来讲一下我们的势能函数，我们定义：

ϕ(x)=log⁡∣x∣Φ(T)=∑x∈Tϕ(x)\phi(x) = \log \left|x\right|\\\Phi(T) = \sum_{x \in T}\phi(x)ϕ(x)=log∣x∣Φ(T)=x∈T∑​ϕ(x)

可以发现，对于任意时刻，因为 ∣x∣≥1\left|x\right| \ge 1∣x∣≥1，因此 ϕ(x)≥0\phi(x) \ge 0ϕ(x)≥0，从而得到 Φ(T)≥0\Phi(T) \ge 0Φ(T)≥0，因此势能函数是合法的。
同时 ∀∣x∣≤∣T∣\forall\left|x\right|\le\left|T\right|∀∣x∣≤∣T∣，因此我们总有 ϕ(T)≤∣T∣log⁡∣T∣\phi(T) \le \left|T\right|\log\left|T\right|ϕ(T)≤∣T∣log∣T∣。这个上界是比较松的，但是对我们的分析没有影响。

下面考虑一次伸展操作对于势能函数的影响。

由于我们可以把从根向下查找的代价计算到伸展过程中对应的旋转操作上，此时旋转操作复杂度不变，只是常数增大，从而忽略了查找对复杂度的影响。
我们可以简单地通过增大势的单位来支配隐藏在操作中的常数。
因此我们只需证明对于一次伸展操作的所有旋转操作，其复杂度是均摊 log⁡∣T∣\log \left|T\right|log∣T∣ 的，我们就完成了对 Splay 复杂度的证明。

  1. zig/zag

由于 zig 操作与 zag 相似，因此只需要证明 zig 即可。

假设我们 zig 的对象是 xxx，其父亲为 yyy，显然在旋转以后，只有 xxx 和 yyy 的子树大小发生了变化。
因此势能变化量为：

ΔΦ(T)=ϕ(x′)+ϕ(y′)−ϕ(x)−ϕ(y)\Delta\Phi(T) = \phi(x') +\phi(y') - \phi(x) - \phi(y)ΔΦ(T)=ϕ(x′)+ϕ(y′)−ϕ(x)−ϕ(y)

显然 ϕ(x′)=ϕ(y)\phi(x') = \phi(y)ϕ(x′)=ϕ(y)，且 ϕ(x′)≥ϕ(y′)\phi(x')\ge\phi(y')ϕ(x′)≥ϕ(y′)，因此消去 ϕ(x′)\phi(x')ϕ(x′) 与 ϕ(y)\phi(y)ϕ(y)，并将 ϕ(y′)\phi(y')ϕ(y′) 替换为 ϕ(x′)\phi(x')ϕ(x′)，有：

ΔΦ(T)≤ϕ(x′)−ϕ(x)\Delta\Phi(T) \le \phi(x') - \phi(x)ΔΦ(T)≤ϕ(x′)−ϕ(x)

因此 zig 操作的均摊代价为 O(1+ϕ(x′)−ϕ(x))O(1 + \phi(x') - \phi(x))O(1+ϕ(x′)−ϕ(x))，其中 O(1)O(1)O(1) 代表旋转操作本身的复杂度。
而在一次伸展操作中也只会有一次 zig 操作，因此这额外的 O(1)O(1)O(1) 代价不会对分析造成影响，因此我们可以只关心其中的 O(ϕ(x′)−ϕ(x))O(\phi(x') - \phi(x))O(ϕ(x′)−ϕ(x))。

  1. zig-zig/zag-zag

由于 zag-zag 操作与 zig-zig 相似，因此只需要证明 zig-zig 即可。

假设我们 zig-zig 的对象是 xxx，其父亲为 yyy，其祖父为 zzz，与 zig 操作类似，势能变化量为：

ΔΦ(T)=ϕ(x′)+ϕ(y′)+ϕ(z′)−ϕ(x)−ϕ(y)−ϕ(z)\Delta\Phi(T) = \phi(x') + \phi(y') + \phi(z') - \phi(x) - \phi(y) - \phi(z)ΔΦ(T)=ϕ(x′)+ϕ(y′)+ϕ(z′)−ϕ(x)−ϕ(y)−ϕ(z)

同样地，由于 ϕ(x′)=ϕ(z)\phi(x') = \phi(z)ϕ(x′)=ϕ(z)，因此将它们消去：

Δϕ(T)=ϕ(y′)+ϕ(z′)−ϕ(x)−ϕ(y)\Delta\phi(T) = \phi(y') + \phi(z') - \phi(x) - \phi(y)Δϕ(T)=ϕ(y′)+ϕ(z′)−ϕ(x)−ϕ(y)

而我们又有 ϕ(x′)≥ϕ(y′),ϕ(x)≤ϕ(y)\phi(x') \ge \phi(y'), \phi(x) \le \phi(y)ϕ(x′)≥ϕ(y′),ϕ(x)≤ϕ(y)，因此有：

ΔΦ(T)≤ϕ(x′)+ϕ(z′)−2ϕ(x)\Delta\Phi(T) \le \phi(x') + \phi(z') - 2\phi(x)ΔΦ(T)≤ϕ(x′)+ϕ(z′)−2ϕ(x)

推到这里，我们先来做一个小工作，来证明 ϕ(x)+ϕ(z′)−2ϕ(x′)\phi(x) + \phi(z') - 2\phi(x')ϕ(x)+ϕ(z′)−2ϕ(x′)（注意与上面的式子不一样）的值不大于 −1-1−1。

假设 ∣x∣=a,∣z′∣=b\left|x\right| = a, \left|z'\right| = b∣x∣=a,∣z′∣=b，那么我们有：

ϕ(x)+ϕ(z′)−2ϕ(x′)=log⁡∣x∣+log⁡∣z′∣−2log⁡∣x′∣\phi(x) + \phi(z') - 2\phi(x') = \log\left|x\right|+\log\left|z'\right| - 2\log\left|x'\right|ϕ(x)+ϕ(z′)−2ϕ(x′)=log∣x∣+log∣z′∣−2log∣x′∣

我们将 log⁡\loglog 合并，得到：

ϕ(x)+ϕ(z′)−2ϕ(x′)=log⁡(∣x∣∣z′∣∣x′∣2)\phi(x) + \phi(z') - 2\phi(x') = \log(\dfrac{\left|x\right|\left|z'\right|}{\left|x'\right|^2})ϕ(x)+ϕ(z′)−2ϕ(x′)=log(∣x′∣2∣x∣∣z′∣​)

由于 ∣x′∣≥a+b\left|x'\right|\ge a+ b∣x′∣≥a+b（可以结合旋转过程思考一下），而 log⁡\loglog 是单调的，因此：

ϕ(x)+ϕ(z′)−2ϕ(x′)≤log⁡(ab(a+b)2)≤log⁡(ab2ab)≤−1\phi(x) + \phi(z') - 2\phi(x') \le \log(\dfrac{ab}{(a + b)^2}) \le \log(\dfrac{ab}{2ab}) \le -1ϕ(x)+ϕ(z′)−2ϕ(x′)≤log((a+b)2ab​)≤log(2abab​)≤−1

证明完毕。
现在我们已经知道 zig-zig 操作的摊还代价不大于：

O(1)+ϕ(x′)+ϕ(z′)−2ϕ(x)O(1) + \phi(x') + \phi(z') - 2\phi(x)O(1)+ϕ(x′)+ϕ(z′)−2ϕ(x)

其中 O(1)O(1)O(1) 为旋转操作的复杂度。由于之前的推导我们可以知道 ϕ(x)+ϕ(z′)−2ϕ(x′)≤−1\phi(x)+\phi(z')-2\phi(x')\le-1ϕ(x)+ϕ(z′)−2ϕ(x′)≤−1，因此 −1−(ϕ(x)+ϕ(z′)−2ϕ(x′))≥0-1-(\phi(x)+\phi(z')-2\phi(x'))\ge 0−1−(ϕ(x)+ϕ(z′)−2ϕ(x′))≥0，我们在摊还代价上加上这个非负数得到：

O(1)+ϕ(x′)+ϕ(z′)−2ϕ(x)−1−(ϕ(x)+ϕ(z′)−2ϕ(x′))O(1) + \phi(x') + \phi(z') - 2\phi(x) - 1-(\phi(x)+\phi(z') - 2\phi(x'))O(1)+ϕ(x′)+ϕ(z′)−2ϕ(x)−1−(ϕ(x)+ϕ(z′)−2ϕ(x′))

化简一下，就得到：

O(1)+O(ϕ(x′)−ϕ(x))−1O(1) + O(\phi(x') - \phi(x)) - 1O(1)+O(ϕ(x′)−ϕ(x))−1

通过增大我们刚刚加的那个非负数以及势的单位，我们就可以支配隐藏在 O(1)O(1)O(1) 中的常数，因此一次 zig-zig 操作的摊还代价为：

O(ϕ(x′)−ϕ(x))O(\phi(x') - \phi(x))O(ϕ(x′)−ϕ(x))

  1. zig-zag/zag-zig

分析的过程和 zig-zig 操作完全一样，之前分析用到的所有性质此时仍然适用，因此略过分析过程。其摊还代价依然为：

O(ϕ(x′)−ϕ(x))O(\phi(x') - \phi(x))O(ϕ(x′)−ϕ(x))

总结：
综上所述，除了最后一次旋转可能增加 O(1)O(1)O(1) 的代价以外。
其余操作的摊还代价只和我们伸展的对象 xxx 的势能有关。
我们假设旋转操作一共执行了 nnn 次，并用 xix_ixi​ 来表示节点 xxx 在经过 iii 次旋转后的状态，那么整一个伸展操作的摊还代价就为：

O(1+∑i=1nϕ(xi)−ϕ(xi−1))O(1 + \sum_{i = 1} ^n\phi(x_i) - \phi(x_{i - 1}))O(1+i=1∑n​ϕ(xi​)−ϕ(xi−1​))

显然除了 ϕ(xn)\phi(x_n)ϕ(xn​) 与 ϕ(x0)\phi(x_0)ϕ(x0​) 外，所有的势能都被抵消了，因此摊还代价为：

O(1+ϕ(xn)−ϕ(x0))O(1 + \phi(x_n) - \phi(x_0))O(1+ϕ(xn​)−ϕ(x0​))

至此，我们不必关心 ϕ(x0)\phi(x_0)ϕ(x0​) 的值了。
此时 ϕ(xn)\phi(x_n)ϕ(xn​) 是 TTT 的根，因此 ϕ(xn)=log⁡∣T∣\phi(x_n) = \log\left|T\right|ϕ(xn​)=log∣T∣。
我们成功的证明了一次伸展操作的摊还代价为 log⁡∣T∣\log \left|T\right|log∣T∣。

所以是均摊 O(log⁡n)O(\log n)O(logn)。
